Melhores abordagens para implementação

  • 1 Screenshot de página inteira (recomendado): usar uma captura full-page (ex: extensão GoFullPage ou Playwright) em vez do thumbnail atual — a imagem ideal tem ~1920×7000 px. Com uma imagem longa, o scroll do mockup percorre seções reais do site.
  • 2 Mouse-scroll interativo (este teste): cursor Y → posição no screenshot. Lerp suave no JS. Zero dependências, funciona offline, sem CORS. É a abordagem mais robusta para portfólio.
  • 3 Auto-scroll com pausa no hover: scroll contínuo lento quando idle, controle pelo mouse quando o usuário entra. Já implementado neste teste — dá vida ao card mesmo sem interação.
  • 4 iframe embed (alternativa): incorporar o site real via iframe para interação completa. Problema: maioria dos sites bloqueia com X-Frame-Options. Aluga Aqui provavelmente bloqueia. Não recomendado para produção.
  • 5 Click-to-expand lightbox: card menor + clique abre modal com o mockup em tamanho maior. Boa alternativa para a grade de 3 colunas sem alterar o layout existente.
